Just a few highlights of Codio's IDE features...

More features than hairs on Chuck Norris chest!

Abbreviations read the docs

There is a very wide range of shortcuts available for HTML, CSS and Javascript. You simply enter the abbreviation in the Code Editor and then press the Tab key to expand the abbreviation. Take a look at the example below.

Command Bar read the docs

The Command Bar lets you access almost every Codio command with a single keyboard shortcut. You simply type the command you want to run and it autocompletes the list for you. It's a great place to look up keyboard shortcuts without having to explore the menu or documentation.

The example below shows how a user toggles the line wrapping (which can also be done with a dedicated keyboard shortcut).

Panels & Tabs read the docs

You can create any panel configuration you want for both code editing and also other features (such as SSH, Bower, Deployment etc.).

This gives you complete control over the layout of your project. Whenever you leave you project, the panel configuration is saved and reloaded when you come back to it.

You can open multiple tabs within a panel. You can drag tabs from one panel to another and you can switch tabs using the keyboard using `Alt+Shift+>` and `Alt+Shift+<`

Keyboard Shortcuts read the docs

Codio has over 80 actions that can be invoked with a keyboard shortcut. If you are used to Sublime, Vim or Textmate then you won't be disappointed.

All keyboard shortcuts can be overriden in the Settings screen.

Quick Open read the docs

Quick Open is a great way of opening files without the need to access the file tree. You can start typing the name of a file and as you type, the list will narrow down. You can then select the file you want using the mouse or use the keyboard.

Beautify read the docs

Beautify your HTML, CSS and JS code (entire file or selected block) with a single key press. You customize over 20 individual settings to get the exact format you're after.

Linting & Folding read the docs

Codio is full of really powerful little features. The example animation below shows a couple of these - code-folding and linting - in action.

Bower read the docs

We've built Bower right into the IDE to make things a bit easier for you. You can search through Bower packages and then click to install. You can also update and remove your Bower components.

Templates read the docs

Codio provides a template management system so when you create new files of differeing types, a code stub us automatically inserted into the new file. There various tokens such as date, time, project name etc. Everything is editable and you can add your own.

Import read the docs

Create new projects from templates or import them from Git, Mercurial, (S)FTP or ZIP files.

SSH read the docs

The SSH manager allows you to create and manage SSH connections to any number of remote servers. You can even upload your Codio SSH key straight to the remote server with a single button press.

Deployment read the docs

The Deployment manager lets you create and manage any number of remote deployment targets. We currently support Git, FTP, SFTP, RSYNC, Nodejitsu and Parse targets.

Once set up, you can deploy to your deployment targets with a single button click. Take a look at the animation below.

And some other cool features...

More features than hairs on Chuck Norris chest!

Autosave

You don't need to worry about saving manually. Codio does it for you. You can use undo and Git/Mercurial to take care of all version control requirements.

Syntax Highlighting

Syntax highlighting is applied to each file automatically. You can also manually apply highlighting from a list of over 80 formats

Forking

You can fork any Public project from any Codio user. Private projects are only forkable by those with explicit permissions.

README.md

Codio has support for README.md files so anyone opening up your project will automatically see an overview (a la Github).

Drag & Drop Files

Chrome users are able to drag and drop files from their desktop right into the Codio IDE at any point you require in the File Tree..
