Just a few highlights of Codio's Back End features...

More features than hairs on Chuck Norris chest!

Every project gets a Box read the docs

It couldn't be easier. Each Codio project gets a Box, out of the box. Nothing to configure. No delays and no hibernation. Our base Box is fast and free to use for all public projects.

It comes preconfigured with Node (with full NVM support) and Ruby (with full RVM support).

Nodetastic

Clone your Node project (or create a brand new one), run `npm install` and you're ready to start coding within the Codio IDE. Even better, our integrated Node debugger will be out soon.

Codio comes with nvm installed, so you can run any node version you like.

Rubylicious

Ruby is also ready to run and, as with Node projects. Clone your project into Codio and you're ready to run and our rbenv support means any project will work just fine.

Run Commands read the docs

You can configure your own shell commands so they run with a click within Codio. Saves you the hassle of having to enter common commands in the Terminal window.

All keyboard shortcuts can be overriden in the Settings screen. Take a look at the animation below.

Node Debugger

From March, we'll be be releasing a Node debugger that will include breakpoint, stepping and variable & call-stack inspection.

Bigger Boxes

Currently, there we offer a single box size. Before long, however, we will be offering a choice of box sizes.

Always-On Boxes

We will soon be offering Boxes that run whether your project is open in the Codio IDE or not, so you can leave server side applications running all the time.
